# Crypto Investigation Workflow

This document outlines the investigation workflow used to analyze blockchain wallet activity and transaction behavior.

---

## 1. Define the Investigation Target

The investigation starts with a target such as:

- Wallet address
- Transaction hash
- Token contract
- Smart contract address
- Suspicious dApp
- Address connected to a scam or exploit

---

## 2. Classify the Address

The target address is classified as one of the following:

- Externally Owned Account, also known as a user wallet
- Smart contract
- Exchange wallet
- Token contract
- Protocol contract
- Unknown or unlabeled address

---

## 3. Review Wallet Overview

The wallet overview includes:

- Current ETH balance
- Token holdings
- Transaction count
- First transaction date
- Most recent transaction date
- Known labels
- Wallet age
- General activity pattern

---

## 4. Review Transactions

Transactions are reviewed for:

- Incoming transfers
- Outgoing transfers
- Failed transactions
- Token transfers
- Internal transactions
- Contract calls
- Large-value transactions
- Repeated behavior patterns

---

## 5. Review Contract Interactions

Contract interactions are reviewed to identify:

- Approvals
- Swaps
- Claims
- Mints
- Transfers
- Bridge activity
- DeFi interactions
- Suspicious contract calls

---

## 6. Identify Risk Indicators

Risk indicators may include:

- Interaction with suspicious contracts
- Unusual approval behavior
- Rapid fund movement
- Repeated failed transactions
- Wallet clustering
- Large outgoing transfers
- Newly created wallets moving funds quickly
- Links to known scam or exploit addresses

---

## 7. Document Evidence

Evidence should be documented clearly using:

- Transaction hashes
- Wallet addresses
- Screenshots
- Timestamps
- Token transfer records
- Contract interaction notes
- Timeline summaries

---

## 8. Write Final Findings

The final report should include:

- Executive summary
- Target overview
- Key findings
- Risk indicators
- Evidence table
- Timeline
- Final conclusion
